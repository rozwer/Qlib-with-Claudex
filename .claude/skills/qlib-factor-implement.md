## Quality Criteria

### Required
- Valid Python syntax
- Produces `result.h5` when executed
- No look-ahead bias (use only past data)
- Handles NaN appropriately

### Target Metrics
- IC > 0.03 is considered good
- Stable IC across time periods indicates robustness

## Common Patterns (successful examples)

1. **Momentum**: `close / close.shift(N) - 1`
2. **Mean Reversion**: `(close - close.rolling(N).mean()) / close.rolling(N).std()`
3. **Volume-Price**: `(close * volume) / (close * volume).rolling(N).mean()`
4. **Volatility**: `close.rolling(N).std() / close.rolling(N).mean()`

## Common Mistakes (avoid)

| Mistake | Fix |
|---------|-----|
| Using future data (look-ahead) | Use `.shift(N)` with N > 0, `.rolling()` only |
| Wrong column names | Use exactly: `open`, `close`, `high`, `low`, `volume`, `vwap` |
| Not handling MultiIndex | Use `.groupby(level="instrument")` for per-stock calculations |
| Invalid output format | Keep the original MultiIndex as a Series |
| Division by zero | Add `.replace([np.inf, -np.inf], np.nan)` |
| Index corruption from `groupby.apply()` | Use `groupby.transform()` instead (apply reconstructs the index and can cause NaN) |
| Using missing columns | Columns with `usable=false` in `data_quality.json` (e.g., vwap) have been pre-validated. Check with `notna().sum() > 0` before use, or use alternative calculations |
